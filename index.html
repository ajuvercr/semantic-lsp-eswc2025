What is the Semantic Web Language Server (SWLS)?

  • Language Server implementation in Rust (Bevy ECS-based)
  • Supports Turtle, JSON-LD, and SPARQL
  • Features: completion, diagnostics, highlighting, formatting, hover

Language Server Protocol

  • Standard JSON-RPC protocol.
  • Reduces integration complexity: O(M×N) becomes O(M+N).
  • Cross-editor language support (VSCode, NeoVim, Jetbrains, etc.).

Entity Component System (ECS)

  • Entities: bundles of components, here documents in the editor
  • Components: objects part of an entity
  • Systems: functions that acts on entities with the desired components

Entity Component System (ECS)

Originally created to handle data management in games, following composition over inheritance, all data is present in the world

In a game an entity might be the protagonist, or enemies, or buildings, or ...

A system might be gravity, which works over all entities with a mass and a velocity.

Conclusion

  • SWLS improves on the state of the art by
    1. being langauge agnostic
    2. enabling local integration
  • Adding a language is as simple as adding a tokenizer and a parser

Future Work

  • Support local ontology files
